why web standards? Web standards make web development easier.

  • HTMLHyperText Markup Language

    HTML은 하이퍼텍스트 마크업 언어(HyperText Markup Language)라는 의미의 웹 페이지를 위한 지배적인 마크업 언어이다. HTML은 제목, 단락, 목록 등과 같은 본문을 위한 구조적 의미를 나타내는 것뿐만 아니라 링크, 인용과 그 밖의 항목으로 구조적 문서를 만들 수 있는 방법을 제공한다. 그리고 이미지와 객체를 내장하고 대화형 양식을 생성하는 데 사용될 수 있습니다.

  • CSSCasscadingStyle Shet

    구조적인 문서(HTML,XML)에 Style(글자, 여백, 레이아웃)을 적용하기 위해 사용하는 언어(Language) 운영체제나 프로그램에 관계없이 누구나 같은 형식으로 보기 위해 만들어진 W3C(World Wide Web Consotium)에서 공인한 문서형식의 표준. CSS를 제거하면 아래의 그림처럼 구조만 남게 되며, CSS를 적용해 주면 위의 그림과 같이 정상적인 상태가 됩니다.

  • javascript & jqueryprogramming Language

    자바스크립트는 객체지향 스크립트 언어로서 프로그램 코드가 HTML문서 사이에 직접 들어가게 된다. 자바스크립트로 만든 프로그램에서는 사용자가 마우스를 이용해 클릭하거나 키보드를 입력하는 것과 같은 작업을 즉시 처리할수 있다. 그렇기 때문에 자바스크립트는 사용자가 입력한 내용이 제대로 되었는지를 서버로 데이터를 전송하지 않고 즉석에서 확인하여 처리할 수 있는 것이다.

webgrid&css typograhy

웹 그리드 시스템 & CSS 타이포그래피는 웹표준의 단조로운 플랫한 스타일을, 조금 더 편집적 느낌으로 보여주는 방식이다.

또한, CSS 방식의 타이포그래피 적용이 핵심이다.

타이포그래피를 적용시키기 위해서는 가독성 요소인 자간 행간 자폭 등이 구성되어야 하며 판독성 요소인 띄어쓰기 및 여백 등의 레이아웃 기술이 필요한 부분이다.

margin : 페이지 공간에서 그리드를 둘러싸고 있는 외곽 공간

column : 정보를 배치하기 위해 세로로 길게 분활된 공간

column interval : 단과 단사이의 간격

flowlines : 수평으로 공간을 가로지르는 시각기준선은 시선의 흐름을 페이지 공간으로 안내하는 역할을 담당

field, module : 단과 열로 나누어진 그리드의 기본단위를 필드

3단그리드는 보편적인 포맷으로 안정감이있고, 위험 부담이 없으며, 6단으로 세분화 할 수 있다. 다단그리드를 사용하면 더 많은 융통성이 생기고, 공간을 창조적으로 활용할 수도 있게 된다.

텍스트와 이미지를 함께 다루는 작업에서는 두 요소 간의 비율을 생각하여 각각 어느 정도의 공간을 할애할 것인지를 결정한다
